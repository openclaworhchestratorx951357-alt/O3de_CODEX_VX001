# Remote Automation Surface Matrix

## Purpose

This is the canonical matrix for admitting remote O3DE automation surfaces into
production scope for this repository.

It is intentionally stricter than a tool catalog. A surface belongs here only if
its production truth, substrate, guardrails, and release evidence can be stated
clearly.

Current branch truth reminder:
- `project.inspect` is real read-only
- `build.configure` is real plan-only
- `settings.patch` is the first admitted mutation-gated real path
- broader real adapter coverage is still narrow
- simulated vs real wording must stay explicit

## Interpretation Rules

- "Current truth" wins over roadmap ambition.
- A surface is not real merely because O3DE officially supports the substrate.
- Mutation-capable rows must meet the same backup/rollback discipline modeled by
  the admitted `settings.patch` path before admission.
- Frontend truth labels must distinguish:
  - real read-only
  - real plan-only
  - real mutation-gated
  - simulated
  - optional later workstream / not admitted
