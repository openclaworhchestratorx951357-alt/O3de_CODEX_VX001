# Remote Automation Surface Matrix

## Purpose

This is the canonical matrix for admitting remote O3DE automation surfaces into
production scope for this repository.

It is intentionally stricter than a tool catalog. A surface belongs here only if
its production truth, substrate, guardrails, and release evidence can be stated
clearly.

Current branch truth reminder:
- `project.inspect` is real read-only
- `build.configure` is real plan-only
- `settings.patch` is the first admitted mutation-gated real path
- `build.compile` is admitted real execution-gated for explicit named targets
- `asset.processor.status`, `asset.source.inspect`, `render.capture.viewport`,
  `render.material.inspect`, and `test.visual.diff` are admitted narrow real
  read-only evidence paths
- `asset.batch.process`, `asset.move.safe`, `render.shader.rebuild`,
  `test.run.gtest`, `test.run.editor_python`, and `test.tiaf.sequence` are
  admitted real plan-only preflight/result-truth paths
- `gem.enable` and `render.material.patch` are admitted narrow mutation-gated
  real paths alongside `settings.patch`
- `editor.session.open`, `editor.level.open`, `editor.entity.create`, and
  `editor.component.add` are admitted real editor-authoring/runtime on the
  verified `McpSandbox` target wiring
- `editor.entity.exists` and `editor.component.property.get` are admitted
  hybrid read-only, and the composed editor chain plus standalone direct
  entity-exists readback were live-proven on `2026-04-25` through repo-owned
  proof harnesses
- broader real adapter coverage is still narrow
- simulated vs real wording must stay explicit

## Release Gate Check

Run the repository-owned matrix drift check before using this document as a
handoff or release-gate input:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1 surface-matrix-check
```

This check is intentionally lightweight. It verifies that this matrix still
names the current admitted surface classes and does not silently fall back to
older "not yet admitted" wording for already-admitted narrow slices.

## Interpretation Rules

- "Current truth" wins over roadmap ambition.
- A surface is not real merely because O3DE officially supports the substrate.
- Mutation-capable rows must meet the same backup/rollback discipline modeled by
  the admitted `settings.patch` path before admission.
- Frontend truth labels must distinguish:
  - real read-only
  - real plan-only
  - real execution-gated
  - real mutation-gated
  - real editor-authoring/runtime
  - simulated
  - optional later workstream / not admitted
